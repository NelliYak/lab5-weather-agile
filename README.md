# Лабораторная работа 5: Моделирование Scrum-процесса
## Проект: "Погода на день" (Weather Today)

Проект представляет собой одностраничный сайт для отображения текущей погоды. Пользователь может ввести название города и получить актуальную информацию о температуре, влажности, скорости ветра и прогнозе на день. Сайт имеет минималистичный дизайн, адаптирован под мобильные устройства и использует открытое API для получения данных о погоде.

## Sprint Review

### Завершенные задачи в спринте:
1. **Настройка базовой структуры проекта** - созданы основные файлы и папки, настроена сборка
2. **Разработка HTML-каркаса** - реализована основная верстка страницы с полем ввода и контейнерами для данных

### Доставленный инкремент:
По итогам спринта мы имеем рабочую HTML-страницу с базовой структурой сайта погоды. Страница корректно отображается в браузере, содержит все необходимые элементы интерфейса (заголовок, поле поиска, блоки для вывода данных). Это основа для подключения API и добавления функциональности в следующих спринтах.

### Незавершенные задачи:
1. **Подключение API погоды** - задача не была завершена из-за сложностей с получением API-ключа и настройкой CORS. Переносим в следующий спринт.

## Sprint Retrospective

### Что прошло хорошо:
1. Четкое распределение задач и ролей в команде
2. Быстрое создание базовой структуры проекта
3. Эффективное использование инструментов GitHub для отслеживания прогресса
4. Соблюдение дедлайнов по первым двум задачам

### Что пошло не так:
1. Не учли время на регистрацию и настройку стороннего API
2. Отсутствовала резервная копия плана на случай проблем с API
3. Недостаточно детально проработали критерии готовности для задачи с API

### Что можно улучшить в следующем спринте:
1. Добавить этап исследования API до включения задачи в спринт
2. Создать заглушки (mock data) для тестирования интерфейса без работающего API
3. Разбить крупные задачи на более мелкие подзадачи
4. Выделить время на ежедневные стендапы для оперативного решения проблем
